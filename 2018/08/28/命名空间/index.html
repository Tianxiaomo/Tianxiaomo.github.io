为什么存在命名空间

随着项目的增大，名称相互冲突的可能性也再增加，不同成员、不同厂商的命名可能出现冲突，不兼容等问题。为了解决这种冲突，命名空间出现了。

默认存在命名空间

声明区域(declaration region)

可以在其中声明的区域

  • 全局声明：在函数外部声明
  • 局部声明：在函数内部声明

    潜在作用区域

    从声明点开始 –> 声明区域末尾


自定义的命名空间

自定义命名空间，提供一个声明名称的区域，一个命名空间中的名称不会与另一个命名空间中的名称冲突，同时其他部分可以使用命名空间中的东西。关键字：namespace

using 声明 v.s using 编译指令
